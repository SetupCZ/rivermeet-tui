For persistent configuration, you can create a JSON config file. Environment variables will override any values set in the config file.

## Configuration Priority

Rivermeet TUI loads configuration in this order (highest to lowest priority):

1. **Environment variables** (highest priority) - Always override everything
2. **Config file** - Overrides defaults
3. **Built-in defaults** (lowest priority)

<Aside type="tip">
  This means you can set defaults in your config file and override specific values with environment variables when needed (e.g., for different Confluence instances).
</Aside>

## Config File Location

The config file should be placed at:

```
~/.config/rivermeet-tui/config.json
```

Or if you have `XDG_CONFIG_HOME` set:

```
$XDG_CONFIG_HOME/rivermeet-tui/config.json
```

<FileTree>
- .config/
  - rivermeet-tui/
    - config.json
</FileTree>

## Full Configuration Example

```json
{
  "confluence": {
    "baseUrl": "https://your-domain.atlassian.net",
    "email": "your-email@example.com",
    "apiToken": "your-api-token"
  },
  "editor": "vim",
  "cacheDir": "~/.cache/rivermeet-tui",
  "keyBindings": {
    "up": ["k", "up"],
    "down": ["j", "down"],
    "left": ["h", "left"],
    "right": ["l", "right"],
    "select": ["return", "enter"],
    "back": ["escape", "backspace"],
    "halfPageUp": ["ctrl+u"],
    "halfPageDown": ["ctrl+d"],
    "lineStart": ["0"],
    "lineEnd": ["$"],
    "documentTop": ["g"],
    "documentBottom": ["G"],
    "wordForward": ["w"],
    "wordBackward": ["b"],
    "edit": ["i"],
    "quit": ["q"],
    "search": ["/"],
    "searchNext": ["n"],
    "searchPrev": ["N"],
    "refresh": ["r"],
    "publish": ["p"],
    "yank": ["y"],
    "visualChar": ["v"],
    "visualLine": ["V"],
    "confirm": ["y"],
    "cancel": ["n", "escape"],
    "debug": ["d"]
  },
  "theme": {
    "primary": "#7aa2f7",
    "secondary": "#bb9af7",
    "background": "#1a1b26",
    "text": "#c0caf5",
    "border": "#565f89",
    "highlight": "#3b82f6",
    "error": "#f7768e",
    "warning": "#e0af68"
  }
}
```

## Configuration Options

### confluence

Confluence API connection settings:

| Property | Type | Description |
|----------|------|-------------|
| `baseUrl` | string | Your Confluence instance URL |
| `email` | string | Your Atlassian account email |
| `apiToken` | string | Your Atlassian API token |

<Aside type="caution">
  Storing credentials in the config file means they're saved as plain text. Consider using [environment variables](../environment/) for credentials, which take priority over config file values.
</Aside>

### editor

The text editor to use when pressing `i` to edit a page:

```json
{
  "editor": "nvim"
}
```

Common values: `vim`, `nvim`, `nano`, `code --wait`

<Aside type="note">
  The `EDITOR` environment variable takes priority over this setting.
</Aside>

### cacheDir

Directory for cached pages:

```json
{
  "cacheDir": "~/.cache/rivermeet-tui"
}
```

### keyBindings

Customize keyboard shortcuts. Each action can have multiple keys:

```json
{
  "keyBindings": {
    "up": ["k", "up"],
    "down": ["j", "down"],
    "quit": ["q", "ctrl+c"]
  }
}
```

Key binding format:
- Simple keys: `"k"`, `"/"`, `"0"`
- Arrow keys: `"up"`, `"down"`, `"left"`, `"right"`
- Special keys: `"return"`, `"enter"`, `"escape"`, `"backspace"`
- Modifier combinations: `"ctrl+u"`, `"ctrl+d"`
- Shift keys: Use uppercase letter `"G"` or `"shift+g"`

Available actions:

**Navigation:**

| Action | Default Keys | Description |
|--------|--------------|-------------|
| `up` | `k`, `up` | Move cursor up |
| `down` | `j`, `down` | Move cursor down |
| `left` | `h`, `left` | Move left / collapse |
| `right` | `l`, `right` | Move right / expand |
| `select` | `return`, `enter` | Select / open |
| `back` | `escape`, `backspace` | Go back / cancel |

**Page Navigation:**

| Action | Default Keys | Description |
|--------|--------------|-------------|
| `halfPageUp` | `ctrl+u` | Scroll half page up |
| `halfPageDown` | `ctrl+d` | Scroll half page down |
| `lineStart` | `0` | Go to start of line |
| `lineEnd` | `$` | Go to end of line |
| `documentTop` | `g` | Go to top of document |
| `documentBottom` | `G` | Go to bottom of document |
| `wordForward` | `w` | Move forward one word |
| `wordBackward` | `b` | Move backward one word |

**Actions:**

| Action | Default Keys | Description |
|--------|--------------|-------------|
| `edit` | `i` | Edit page in editor |
| `quit` | `q` | Quit application |
| `search` | `/` | Start search |
| `searchNext` | `n` | Jump to next search match |
| `searchPrev` | `N` | Jump to previous search match |
| `refresh` | `r` | Refresh spaces |
| `publish` | `p` | Publish pending changes |
| `yank` | `y` | Copy selection/line to clipboard |
| `visualChar` | `v` | Enter character visual mode |
| `visualLine` | `V` | Enter line visual mode |
| `confirm` | `y` | Confirm action |
| `cancel` | `n`, `escape` | Cancel action |
| `debug` | `d` | Toggle debug panel (debug mode only) |

### theme

Customize the color scheme:

```json
{
  "theme": {
    "primary": "#7aa2f7",
    "secondary": "#bb9af7",
    "background": "#1a1b26",
    "text": "#c0caf5",
    "border": "#565f89",
    "highlight": "#3b82f6",
    "error": "#f7768e",
    "warning": "#e0af68"
  }
}
```

## Minimal Configuration

You only need to specify what you want to override. A minimal config might be:

```json
{
  "editor": "nvim"
}
```

With credentials provided via [environment variables](../environment/).
